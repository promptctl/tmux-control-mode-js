<objective>

Purpose: ENC-01..ENC-04. Without this, the encoder and client formats can drift, escaping can be done twice or not at all, and there is no single place to audit the wire protocol.

Output: Refactored encoder + client + expanded encoder tests. Public API unchanged. No new commands beyond what already exists.
</objective>

Current client.ts inline violations (to be removed):
```typescript
// line 116 — sendKeys
return this.execute(`send-keys -t ${tmuxEscape(target)} -l ${tmuxEscape(keys)}`);
// lines 119-125 — splitWindow
const dirFlag = options.vertical === true ? "-v" : "-h";
const targetPart = options.target !== undefined ? ` -t ${tmuxEscape(options.target)}` : "";
return this.execute(`split-window ${dirFlag}${targetPart}`);
// line 132 — setSize (duplicates refreshClientSize body)
return this.execute(`refresh-client -C ${width}x${height}`);
// line 137 — setPaneAction (duplicates refreshClientPaneAction body)
return this.execute(`refresh-client -A %${paneId}:${action}`);
```

`TmuxClient.execute(command: string)` is the generic raw-command escape hatch: it takes an already-formatted command (without LF) and wraps it in `buildCommand` before sending. The refactor keeps this: encoder functions return the full wire string including LF, but `execute()` expects a command without LF. Two clean options — pick option B:

- **Option B (chosen):** Add a private `sendRaw(wire: string)` that takes an encoder-produced wire string (with LF) and bypasses `buildCommand`, enqueueing into `pending` and calling `transport.send(wire)` directly. Convenience methods call `sendRaw(encoderFn(...))`. `execute(command)` keeps its current behavior for arbitrary user-supplied raw commands (`listWindows`, `listPanes`).

This keeps the public `execute` contract intact (tests rely on it), and makes every convenience method a thin shim over an encoder function.
