# Phase 5 — Demo Web Multiplexer

## Architecture

```
Browser (Vite + React + Mantine + xterm.js)
   │  import types only, no runtime library code
   │
   │  WebSocket (JSON, base64 for pane bytes)
   ▼
Bridge server (Node: ws + http)
   │  imports TmuxClient
   │
   │  spawnTmux() → -C mode
   ▼
host tmux server
```

## Wire protocol (shared/protocol.ts)

**Browser → server:**
- `{ kind: "execute", id: string, command: string }` — run any tmux command via `client.execute`
- `{ kind: "sendKeys", id: string, target: string, keys: string }` — shortcut for send-keys
- `{ kind: "detach", id: string }` — detach the client

**Server → browser:**
- `{ kind: "event", event: SerializedTmuxMessage }` — any tmux event; `output`/`extended-output` have `data` as base64 string instead of Uint8Array
- `{ kind: "response", id: string, response: CommandResponse }` — reply to execute/sendKeys
- `{ kind: "error", id?: string, message: string }` — bridge-level or protocol error
- `{ kind: "ready" }` — sent after the TmuxClient handshake completes
